# Project Structure & 3-Layer Architecture

This project follows the **Antigravity 3-Layer Architecture** for reliable, deterministic AI automation.

## 🏛️ The 3 Layers

1.  **Layer 1: Directives** (`/directives/`)
    - **Purpose**: Standard Operating Procedures (SOPs) written in Markdown.
    - **Usage**: The Agent reads these to understand *what* to do and *which* tools to use.
    - **Files**: `download_videos.md`, `generate_hover_clips.md`.

2.  **Layer 2: Orchestration** (The AI Agent)
    - **Purpose**: Intelligent routing and decision-making.
    - **Usage**: Connects human intent (Directives) with technical execution (Scripts).

3.  **Layer 3: Execution** (`/execution/`)
    - **Purpose**: Deterministic Python scripts for data processing and API interaction.
    - **Usage**: Called by the Agent to perform the heavy lifting.
    - **Scripts**: `download_videos.py`, `generate_hover_clips.py`.

---

## 📂 Directory Map

- **`/app/`**: The React/Vite frontend application (React, Framer Motion, CSS Modules).
- **`/videos/`**: Local cache for high-resolution source videos from Vimeo/YouTube.
- **`/app/public/hover-clips/`**: Optimized 6-second silent previews used in the UI.
- **`/app/public/thumbnails/`**: Poster images for video portfolio items.
- **`/app/src/data/portfolio.js`**: The single source of truth for portfolio metadata, automatically generated by `execution/generate_hover_clips.py`.
- **`/.tmp/`**: Temporary intermediate files; safe to delete and regenerate.
- **`/.env`**: Sensitive API keys and environment variables (should be Git ignored).
- **`/SKILLS/`**: Specialized agent capabilities (Brand Extraction, Frontend Design, etc.).

## 🎓 Best Practices
- **Deterministic First**: If a task can be coded in Python, code it in `/execution/`. Don't let the LLM handle raw data processing.
- **Directives as SOPs**: Treat directives as training manuals for a human employee. Keep them clear and updated.
- **Cloud Deliverables**: Final outputs should be accessible (Google Sheets/Slides) where appropriate. Local files are for processing.
